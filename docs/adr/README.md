# ADRs (Architecture Decision Records)

ADRs capture non-obvious, durable decisions so future contributors and coding
agents don't invent rationale.

Format (keep it short):

1. Status
2. Context
3. Decision
4. Consequences

If a decision changes, do not rewrite history. Mark the old ADR “Superseded” and
add a new ADR that links back.
