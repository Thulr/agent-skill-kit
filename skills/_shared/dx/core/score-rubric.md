# DX Score Rubric

Rate the current developer experience 0–10 for the target developer and task.
Target is always 10/10.

- **10:** The paved path is obvious, fast, verifiable, debuggable, and safe.
- **8–9:** Strong DX with minor friction, missing edge docs, or small clarity gaps.
- **6–7:** Usable but depends on prior context, source reading, or trial and error.
- **4–5:** Developers can succeed only with help, tribal knowledge, or repeated debugging.
- **1–3:** Setup, integration, or recovery routinely fails.
- **0:** The intended developer task cannot be completed.

Always state the current score, the target score (10), and the minimum changes
needed to reach 10/10.
