# skill-curator Activation Cases

## Positive

- "Turn this book into a skill proposal" -> loads research, pack-placement, depth, and candidate-plan routing.
- "Create a flat skill from this article" -> routes to `create-flat`, loads the public-skill playbook and flat shape, and still emits the required eval artifacts.
- "Create a two-level routing skill like dx-heuristics" -> routes to `create-two-level` and uses the two-level shape.
- "Is this skill flat or hub-and-spoke?" -> routes to `pick-shape` and loads only the depth rubric plus shape references.

## Negative

- "Review this draft skill for source safety" -> use `skill-reviewer`, not `skill-curator`.
- "Install a skill from GitHub" -> use `skill-installer`, not `skill-curator`.
- "Audit this repository's AGENTS.md" -> use `project-agentification`, not `skill-curator`.

## Boundary

- Public skills generated by this repo-local skill use `status: "published"` and carry the required `evals/run-static-checks.sh`, `evals/trigger-evals.json`, and `evals/activation-cases.md`; repo prerelease maturity belongs to the repository tag.
