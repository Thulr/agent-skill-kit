# Artifact templates

Skeletons for the files the `harden-repo-for-coding-agents` `scaffold` intent
produces. Organized by surface (instruction-surface, gates, governance,
skills, reflection-log).

## How these differ from `templates/*.md`

The four templates at `templates/` (`assess-report.md`, `harden-recommendation.md`,
`scaffold-bundle.md`, `diagnose-runbook.md`) are **the skill's own output
templates** — what the skill emits when it runs. They are *not* artifacts
the skill writes to the target repo.

Files under `templates/artifacts/<surface>/` are **artifact templates** —
skeletons the skill copies into the target repo as part of a `scaffold` run.
Each artifact template is referenced from at least one `scaffold`-tier
heuristic in `references/playbooks/<surface>.md`.

## W9 compliance

Templates are **scaffolding**, not autogenerated content. Every template:

- Uses `<placeholder>` markers for required fills. Leaving placeholders
  in the scaffolded file is a static-check failure — the writer has to
  fill them with project-specific content.
- Refuses to be the source of "plausible boilerplate" the W9 warning
  guards against — every section in every template names a specific
  project-knowledge input that must be supplied, not a generic prose
  pattern.
- Optionally cites reflection-log entries (per the
  `rules-from-coding-agent-failures` skill) where a template includes a
  rule derived from a recurring observed failure. This citation is
  optional, not a precondition.

## How the `scaffold` workflow uses these

When `scaffold` produces a file, the scaffold-bundle output's "Proposed
files" table includes a **Template** column citing the path under this
directory. The post-write auditor (step 8) treats an artifact as
`applied` only when:

1. The file exists at the target path.
2. No `<placeholder>` markers remain.
3. The shape matches the template (top-level sections, required frontmatter
   fields, table columns).

## Layout

```
templates/artifacts/
├── README.md                                    (this file)
├── instruction-surface/
│   ├── AGENTS.md                                (WHAT / WHY / HOW skeleton)
│   ├── README-agents-section.md                 (pointer block for repo READMEs)
│   └── check-instruction-surface.sh             (symlink-drift static check)
├── reflection-log/
│   ├── README.md                                (directory index + recording-bar / promotion-bar callout)
│   └── _template.md                             (per-entry template; contributors copy to YYYY-MM-DD-<slug>.md)
├── gates/
│   ├── pretooluse-hook.py                       (argv-parsed deny-list)
│   ├── pretooluse-hook-test.py                  (variant-matrix test fixture)
│   ├── claude-settings.json                     (hook registration block)
│   └── ci-static-checks.yml                     (path-based, all-lanes)
├── governance/
│   ├── CODEOWNERS                               (handle format + lane coverage)
│   └── SECURITY.md                              (disclosure-path skeleton)
└── skills/
    ├── skill.json                               (shape with required fields)
    └── trigger-evals.json                       (canonical schema)
```
