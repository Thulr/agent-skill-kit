# Score rubric (0–10) — REVIEW only

One integer per surface, summarizing how well an agent-native doc serves an agent that acts on it
alone: curated and discoverable, chunk-survivable, budgeted, single-sourced, and triggering
correctly. The audit report carries one score per surface plus a brief justification citing the
playbook's heuristics.

| Score | Band | Meaning |
|------:|------|---------|
| 9–10 | **Exemplary** | Could teach from it. Hand-curated AGENTS.md with synced mirrors and evidence-backed rules; a curated index with placement-by-access-pattern; tool descriptions that disambiguate; chunk-survivable reference with stable anchors; tiered context budget; invariants in gates, not prose. |
| 7–8 | **Healthy** | Minor gaps, documented and contained. No autogenerated AGENTS.md, no safety rule stranded in prose, no everything-dump. |
| 5–6 | **Mixed** | Recognizable but several Medium issues — chunk amnesia, drifting mirrors, a bloated always-loaded file, vague triggers. Cleanup pays off this quarter. |
| 3–4 | **Eroded** | Autogenerated or persona-buried rules, wrong-tool descriptions, or everything-dumps are pervasive; High-severity issues compound; agents act on misleading docs. |
| 1–2 | **Tangled** | The docs mislead the agent — safety invariants in ignorable prose, forked drift, no curation. Critical findings present. |
| 0 | **Absent** | No identifiable agent-native doc surface; agents crawl and guess. |

## How to pick a score

- Start at 5. Move up for each health signal (hand-curated synced AGENTS.md, evidence-backed
  rules, curated index, disambiguated tool descriptions, chunk-survivable + stable-anchored
  reference, tiered budget, invariants-in-gates). Move down for each Critical or High finding.
- A single Critical finding (a safety invariant left in ignorable prose) caps the score at 4.
- Two or more High findings cap the score at 6.
- The score is a summary, not a sum — multiple Low findings should not push a healthy surface
  below 7 by themselves.

## Cross-surface comparison

Scores are comparable only within the same REVIEW run, using the same persona and project
tier. Do not benchmark across teams or across time without recalibrating against the rubric
anchors in `severity-rubric.md`.
