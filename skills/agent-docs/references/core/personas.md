# Target personas

Pick the persona that best matches the audience for this output. The choice shapes vocabulary,
depth, and which trade-offs to surface. The audience here writes the docs an **agent reads and
acts on** — not human docs (that is `docs-audit` / `docs-design`).

## Persona A — Engineer writing an agent-native doc mid-change (DO)

Writing or fixing an `AGENTS.md`, an `llms.txt`, a tool description, or a reference page right
now, and wants an agent to find and act on it correctly without over-building. Comfortable with
their stack; not looking for a docs-theory lecture.

- **Speak to:** hand-curating AGENTS.md, a curated index that names what each link is for, a
  chunk that survives retrieval, a trigger description that disambiguates, the smallest
  always-loaded footprint.
- **Avoid:** an everything-dump context file; a safety invariant left in prose; ceremony for a
  one-page fix.

## Persona B — Docs/DX lead auditing the agent-native doc surface (REVIEW)

Owns a judgment about whether agents can find, choose, and act on the docs: is AGENTS.md
hand-curated and synced, do tools disambiguate, do chunks survive retrieval, is the budget tiered.
Wants scored findings and a short "fix three first" list, not a re-platforming.

- **Speak to:** autogenerated/persona-buried rules, wrong-tool descriptions, chunk amnesia,
  everything-dumps, forked drift, under-triggering, mirror drift.
- **Avoid:** prescribing a vendor; treating every gap as a blocker regardless of project scale
  (calibrate).

## Persona C — Architect designing the agent doc strategy (DESIGN)

Deciding the minimal agent-native doc surface: the AGENTS.md contract, the curated index, the
tool-description conventions, the chunk/anchor discipline, the context-budget tiers. Wants the
smallest doc surface that earns its place, with clear trade-offs.

- **Speak to:** AGENTS.md as a contract with mirrors and evidence-backed rules,
  placement-by-access-pattern, the three load-budget tiers, single-source with drift checks,
  invariants-in-gates not prose.
- **Avoid:** speculative coverage; an everything-dump; treating docs as a reliability fix.

## Persona D — Lead hardening agent docs under deadline

Cannot rewrite the doc set. Needs a sequenced plan to close the trust gaps (de-autogenerate
AGENTS.md, sync mirrors, disambiguate tools, tier the budget) with a safety net at every step.

- **Speak to:** smallest reversible step, moving an invariant from prose to a gate, syncing a
  mirror via symlink/CI before trusting it, what breaks if hardening stops at step N.
- **Avoid:** "first re-author every page"; sequences that block the doc pipeline.

## Default persona

When the prompt does not signal a clear audience, assume **Persona A** for DO, **Persona B**
for REVIEW, and **Persona C** for DESIGN, and state the assumption so the reader can redirect.
