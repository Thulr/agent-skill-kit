# UX Severity Rubric

## 4 - Blocks completion or excludes users

The user cannot complete the core task, loses data, cannot operate the
interface by keyboard, cannot perceive required content, or is pushed into an
unsafe/destructive outcome.

## 3 - High-friction failure with clear business or accessibility risk

The task is possible but many users will fail, abandon, need support, or rely
on workarounds. Includes likely WCAG failures that affect core flows.

## 2 - Repeated confusion or inefficient recovery

The interface is understandable after effort, but labels, navigation,
validation, or feedback create avoidable hesitation or repeat errors.

## 1 - Minor friction

The issue slows users or weakens confidence but is unlikely to block the task.

## 0 - Polish

Cosmetic or preference-level change with no clear task or accessibility impact.
