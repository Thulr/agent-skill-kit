# AGENTS.md Contract Playbook

## Scope

AGENTS.md is the hand-curated, repo-root instruction surface an agent reads first to learn a project: its purpose, layout, commands, invariants, and load-bearing rules. This playbook treats AGENTS.md as a *contract with future agents* — the documentation view of the file — not as the scaffolding that enforces it. The contract covers four things: hand-curation over autogeneration, harness-mirror parity, the >=3-failure provenance floor for load-bearing rules, and the identity-versus-operations boundary that decides where a rule must live to actually transfer.

- **In:** what AGENTS.md should contain and how it stays a contract; the explicit do-not-autogenerate marker; CLAUDE.md / `.github/copilot-instructions.md` / `.cursor/rules` mirror parity; the persona-vs-operations placement rule for safety/ops content.
- **Out:** writing or wiring the hooks/CI that *enforce* invariants (hand off to `harden-repo-for-coding-agents`); the reflection-log workflow that *produces* the >=3 failures a rule cites (hand off to `rules-from-coding-agent-failures`); the typed tool/SDK schema as code (agent-dx owns schema, this surface owns description clarity); generic human README/quickstart (docs-audit/docs-design own those).
- **Intents this surface answers:** do, review, design.

## Grounding

- **AGENTS.md is read by agents before they touch code; it is a contract, not a snapshot.** Hand-curation beats autogeneration because a generated file describes the code as it was, while a curated file states what an agent must know to act safely now — including rules no code line reveals.
- **An explicit "do not autogenerate" marker forestalls drift.** Tools like `/init` or "Generate Cursor Rules" will silently overwrite a curated file with a code snapshot unless the file declares itself off-limits and a gate backs it.
- **Harness mirrors must be one source.** CLAUDE.md, `.github/copilot-instructions.md`, and `.cursor/rules` either symlink to a single AGENTS.md or are CI-checked to match — otherwise each harness drifts to its own dialect.
- **Sub-agents inherit AGENTS.md and SKILL.md but NOT persona/identity files.** A safety or operations rule placed in persona prose silently fails to transfer to spawned sub-agents, so operational invariants must live in AGENTS.md or the harness config.

## Good signals

- AGENTS.md states project purpose, directory layout, the canonical build/test/check commands, and a short list of invariants an agent must not violate.
- The file carries an explicit do-not-autogenerate marker, and a gate fails the build if an autogenerator overwrites it.
- Exactly one source of instruction content exists; CLAUDE.md / `.github/copilot-instructions.md` / `.cursor/rules` are symlinks to it or are CI-checked for parity, with the divergence check named in the file.
- Each load-bearing rule traces to >=3 observed failures (a reflection-log floor) and links its evidence, rather than asserting a generic best practice.
- Safety and operational rules live in AGENTS.md or harness config, where sub-agents inherit them — not in a persona/identity file.
- Rules point to their source of truth (a runbook, schema, or playbook) instead of pasting the full content into the always-loaded file.
- The file is lean: invariant rules and pointers, with expansive detail deferred to load-on-demand references.
- Mandatory invariants are mirrored into deterministic gates (hooks/CI), with AGENTS.md documenting them rather than relying on prose to enforce them.

## Common failures

- AGENTS.md is regenerated by `/init` or "Generate Cursor Rules" and becomes a stale code snapshot, dropping the curated rules that had no code analog.
- A harness mirror (CLAUDE.md, copilot-instructions, cursor/rules) is hand-edited independently and silently diverges from AGENTS.md; no parity check catches it.
- A load-bearing rule is invented from a single incident or a generic best-practice list, producing plausible boilerplate that hurts agent success rather than helping.
- A safety/ops rule lives in a persona file; sub-agents that inherit AGENTS.md/SKILL.md but not the persona never see it, and the rule silently fails to transfer.
- The file is treated as a reliability fix: a mandatory invariant lives only in prose the model may ignore, with no hook/CI gate behind it.
- The everything-dump: layout, full schemas, and long playbooks are pasted inline, bloating always-loaded context and lowering accuracy.
- Stored citations rot because rules link to mutable, unversioned locations instead of stable anchors or source-of-truth pointers.

## Heuristics

- **(do, review, design) Hand-curate; mark do-not-autogenerate.** Author AGENTS.md as a contract with future agents and add an explicit do-not-autogenerate marker, because `/init`-style tools overwrite curated rules with a code snapshot. The marker is documentation; pair it with a gate owned by `harden-repo-for-coding-agents`.
- **(review, design) Keep harness mirrors one source.** Symlink CLAUDE.md, `.github/copilot-instructions.md`, and `.cursor/rules` to a single AGENTS.md, or CI-check them for parity. Name the divergence check in the file so an agent knows drift is gated.
- **(review, design) Gate load-bearing rules on >=3 observed failures.** A rule earns its place by tracing to a reflection-log floor of three or more failures, not by being a plausible best practice. Producing the failures and promoting them is `rules-from-coding-agent-failures`; this surface checks the provenance link exists.
- **(do, review, design) Put operations and safety rules where sub-agents inherit them.** Sub-agents inherit AGENTS.md and SKILL.md but not persona/identity files, so any safety/ops rule in persona prose silently fails to transfer; move it into AGENTS.md or the harness.
- **(design) Don't treat docs as the enforcement layer.** Curation raises average task success, not worst-case reliability. A mandatory invariant must sit in a deterministic gate (hook/CI); AGENTS.md documents the gate, it does not substitute for it.
- **(do, design) Point to source-of-truth, don't paste it.** Keep AGENTS.md to minimal invariant rules and pointers; defer schemas, layout tables, and playbooks to load-on-demand references, because more always-loaded text can lower accuracy, not only raise cost.
- **(review, design) Make every rule a survivable chunk.** Each rule restates its subject, avoids anaphora ("see above"), and links a stable/versioned anchor so a context-bound agent and a stored citation both survive compaction.
- **(do, review) State commands and invariants an agent can act on.** Layout, the one canonical check command, and the short invariant list are the load-bearing payload; if an agent can't run a check or name a forbidden action from the file, the contract is incomplete.

| Are harness mirrors one source (symlink or CI-checked)? | Each harness drifts to its own dialect | Symlink to AGENTS.md or add a parity check |
