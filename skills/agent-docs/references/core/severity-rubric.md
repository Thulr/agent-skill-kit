# Severity rubric (0–4)

Apply to every finding in a REVIEW, every risk in a DESIGN, and every step in a hardening
runbook. Agent-docs severity weighs how badly a doc misleads or fails an agent that acts on it
alone, at machine speed, with no human reading the page.

| Level | Label | Meaning |
|------:|-------|---------|
| 4 | **Critical** | A mandatory safety/destructive-action invariant lives only in markdown a model may ignore (docs-as-reliability-fix) instead of a deterministic gate; AGENTS.md is autogenerated and agents act on confidently-wrong instructions in a destructive context. Block reliance on the doc. |
| 3 | **High** | A safety/operational rule lives in a persona/identity file sub-agents do not inherit; AGENTS.md is autogenerated; tool descriptions are vague/overlapping so the agent calls the wrong tool. Large but tractable fix. |
| 2 | **Medium** | An everything-dump always-loaded file; chunk amnesia (anaphora, no local subject); rotting citations (no stable anchors); forked agent/human docs drifting with no shared source; a skill/resource under-triggering on a vague description; harness mirrors drifting. Fix this cycle. |
| 1 | **Low** | A glossary gap, one missing near-miss exclusion in a trigger description, a slightly verbose index. Queue for cleanup if it accumulates. |
| 0 | **Note** | An observation worth recording, not a defect — e.g. "this doc is human-only by design; agents are routed elsewhere." |

## How to pick a level

1. **Action consequence.** Does a wrong-or-missing doc cause an unsafe/destructive agent action
   (4), a wrong action or wrong tool (3), a degraded/failed retrieval (2), or mere friction (1)?
2. **Enforcement gap.** A mandatory invariant left in ignorable prose rates higher than a
   merely unclear doc — route invariants to gates, not markdown.
3. **Inheritance.** A rule in a file sub-agents do not inherit (persona/identity) silently fails
   to bind; rate it as if the rule were absent.

## Calibration anchors

- "A destructive-command/safety invariant lives only in prose the model may skip" → **4**.
- "AGENTS.md is autogenerated; agents confidently act on wrong instructions" → **3** (4 in a destructive context).
- "A safety/ops rule lives in a persona file sub-agents don't inherit" → **3**.
- "Two tool descriptions are near-identical; the agent calls the wrong one" → **3**.
- "A massive context file is always-loaded; it lowers accuracy and raises cost" → **2** (3 if measurably degrading).
- "Retrieved sections use 'see above'/'this option' with no local subject (chunk amnesia)" → **2**.
- "Stored citations rot because anchors/URLs are not stable" → **2**.
- "Agent docs and human docs are forked with no shared source or drift gate" → **2**.
- "A skill under-triggers because its description omits purpose/use-conditions" → **2**.
- "Harness mirrors (CLAUDE.md vs AGENTS.md) have drifted" → **2**.

## Cross-surface comparison

Scores and severities are comparable only within one REVIEW run, using the same persona and
project tier. Do not benchmark across teams or across time without recalibrating against these
anchors.
