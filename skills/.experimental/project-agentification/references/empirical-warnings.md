# Empirical Warnings

Load-bearing don'ts from the source research. Every playbook that touches one of these patterns should backlink to the relevant anchor.

## W1 — Don't autogenerate AGENTS.md / CLAUDE.md
ETH Zürich / LogicStar.ai (Mündler et al., arXiv:2602.11988, Feb 2026) found that **LLM-generated context files drop task success ~3% on average and inflate cost by over 20%**, while developer-written files improved success by only ~4%. Anthropic's `/init`, Cursor's `/Generate Cursor Rules`, and Codex's auto-init produce plausible but low-value scaffolds. **Hand-curate from observed failures.** The `scaffold` intent of this skill refuses to write any file without at least one stated observed-failure justification.

## W2 — AGENTS.md ≤ 200 lines
Practitioner consensus (HumanLayer, TurboDocx, Augment Code's 2500-repo study): quality degrades sharply above ~300 lines because the harness already injects ~50 system instructions and LLMs follow peripheral instructions more reliably than middle ones. **Push depth into:** hierarchical AGENTS.md per subdirectory, skills, ADRs, and a structured `docs/` tree the agent can pull on demand. HumanLayer's empirical finding: "as you give the LLM more instructions, it doesn't simply ignore the newer ones — it begins to ignore all of them uniformly."

## W3 — Hard gates over soft prose
DEV community write-up on Copilot agent mode: **"CLAUDE.md instructions get followed about 70% of the time; hooks enforce rules at 100%."** Encode constraints as enforced gates (PreToolUse/PostToolUse hooks, CI checks, lint, format-on-write, sandbox modes, MCP allow-lists) — not aspirational prose in instruction files.

## W4 — Don't blindly multi-agent
Walden Yan (Cognition), *Don't Build Multi-Agents* (June 12, 2025): **Principle 1 — share context, and share full agent traces, not just individual messages. Principle 2 — actions carry implicit decisions, and conflicting decisions carry bad results.** Default to a single linear agent. Multi-agent topologies (BMAD, CrewAI, MetaGPT) survive only when handoff artifacts (PRDs, plans, tasks files) re-create shared context across boundaries.

## W5 — AGENTS.md is an attack surface
Prompt Security demonstrated VS Code Chat exfiltration via a malicious AGENTS.md. Skills and hooks are also injection vectors. **Treat AGENTS.md, skills, and hooks as code:** review them, version-control them, write-protect them in the harness (Kilo Code does this by default), and pin reviewing/auditing them to the same bar as production code.

## W6 — Token budget is the dominant scarcity
Anthropic's *Effective Context Engineering for AI Agents*: "find the smallest set of high-signal tokens that maximize the likelihood of your desired outcome." Every recommendation this skill produces must declare its token cost: **always-loaded** (AGENTS.md), **on-trigger** (skill activated by description match), or **on-demand** (MCP method called by agent). Prefer on-demand → on-trigger → always-loaded, in that order.

## W7 — Skills vs AGENTS.md are not interchangeable
Vercel's internal evals: AGENTS.md scored 100% vs 79% for skills on their Next.js benchmark — but only when **hand-curated**. AGENTS.md wins for **global, always-on context**. Skills win for **capability-specific progressively-disclosed workflows**. Putting workflow instructions in AGENTS.md bloats the always-loaded context; putting global conventions in a skill means the agent won't have them when it needs them most.

## W8 — Empirical picture is mixed; ship light and evolve
- ETH/LogicStar: autogenerated AGENTS.md **hurts**.
- Vercel: hand-curated AGENTS.md **wins** over skills (100% vs 79%).
- Anthropic Data Infra team: "more thorough" CLAUDE.md is better on their repetitive tasks.

The synthesis: **hand-curate, keep short, evolve in response to observed failures, prefer hierarchical disclosure over a single long file.** Treat any AGENTS.md / skill / hook as a hypothesis under test, not a finished artifact.

## W9 — Auto-init commands lie
Software Mansion's agentic engineering guide and multiple practitioners: don't use `/init`, `/Generate Cursor Rules`, or equivalents for these files. They produce surface-plausible scaffolds (Simon Willison's "vibe coded outputs are indistinguishable from carefully-engineered ones at a glance") that hide low fitness. **Author by hand against observed failures.**

## W10 — Sandbox isolation matters more than people think
OpenHands docs label process sandboxes "unsafe, but fast" and recommend Docker. OWASP places tool abuse and sandbox escape at the center of agent risk. **Run untrusted agent-generated code inside containers or remote sandboxes by default** — process sandboxes on developer laptops with secrets mounted in writable workspaces is the most common silent-failure mode.
