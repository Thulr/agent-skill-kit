# Audit skills over-produced findings with no project-scale input

## What happened

A user ran `docs-audit` on a small internal project (< 30 files, no external
consumers) and then acted on the audit. The audit recommended creating nearly
as many documentation files as there are code files — the output scaled to the
number of code artifacts, not to the project's stage or stakes.

Root cause: every audit skill (`docs-audit`, `dx-audit`, `ux-audit`,
`architecture-audit`, `perf-audit`, `test-audit`, `writing-audit`) loads a full
heuristic checklist per surface and emits a finding for each gap. Playbook
heuristics are phrased "*every* method / endpoint / flag / empty state / error
needs X" (e.g. `skills/_shared/docs/playbooks/foundations.md` "Source-of-truth
test — identify the canonical artifact for *every* fact class"). Combined with
the four-lens `subagent-dispatch.md` fan-out and the optional `all` multi-surface
mode, a < 30-file prototype gets the same treatment as a 10k-method public
library. Nothing in the workflow takes project size / maturity / stakes as an
input that gates surface breadth, heuristic emission, or finding count. `modes.md`
gates how much the skill *asks*; nothing gated how much it *produces*.

This is not a first observation: memory also records "informed-skills audit-style
skills can surface 50+ issues" (2026-05-19), and
`2026-05-28-restructure-split-justified-by-unenforced-cap.md` was a (rejected)
proposal to *split* skills motivated by them feeling too heavy. The underlying
gap — over-production on small projects — had not been recorded directly.

## What to do differently

Add a cross-cutting calibration layer rather than a hard gate (finding *count*
can't be statically gated; W3 says don't fake a gate you can't enforce):

- New shared reference `skills/_shared/calibration.md` — three tiers
  (Prototype / Growing / Load-bearing), cheap context signals, an infer-then-ask
  rule, and an "every-X collapse" rule (below Load-bearing, collapse
  same-mechanism per-artifact gaps into one systemic finding that inherits the
  max severity — severity rubric and score caps unchanged).
- A terse "Calibrate to project scale" step (4.5) in each audit `SKILL.md`,
  symlinking `references/calibration.md` → `../../_shared/calibration.md`.
- Reinforce output *shape* structurally (W3): report Summary gains a
  `Project tier:` line and "Prioritized fixes" becomes a "Now / Later" split,
  with a static-check grep per skill.

Smallest high-signal response, grounded in literature (barely-sufficient docs;
YAGNI) and the catalog's own W6 (token budget) and W8 (ship light, evolve) —
not an autogenerated rule (W9).
