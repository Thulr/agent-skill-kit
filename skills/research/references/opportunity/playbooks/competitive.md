# Competitive Playbook

## Scope

Who you are actually up against — including the status quo. Maps
direct competitors (same job, same form), substitute competitors
(same job, different form), and the do-nothing alternative
(spreadsheets, email, manual work) which is usually the real
competition. Produces differentiation that survives the do-nothing
comparison, not a feature checklist.

- In: direct / substitute / do-nothing mapping, feature-pricing-
  positioning comparison, moat analysis (brand, ecosystem, data,
  switching costs, network effects), differentiation that's a
  *tradeoff* a competitor can't or won't copy.
- Out: market size (use `market.md`), customer pain (`customer.md`),
  positioning execution at launch (`gtm.md`), trend-driven incumbent
  shifts (`trend.md`).
- Intents this surface answers: scope, investigate, synthesize, decide.

## Grounding

- **Michael Porter — *Competitive Strategy* (1980) / *Five Forces*
  HBR (2008)** — industry attractiveness as a function of rivalry +
  buyer power + supplier power + substitutes + new-entrant threat.
  Used here for the competitive-environment scoring.
- **April Dunford — *Obviously Awesome* (2019)** — positioning as
  deliberate context-setting; competitive alternative isn't a
  product, it's the reference frame the buyer holds you against.
- **Peter Thiel — *Zero to One* (2014)** — sub-segment monopoly
  beats fractional share; differentiation must be a tradeoff
  competitors can't or won't make.
- **Clayton Christensen et al. — *JTBD HBR* (2016)** —
  competing-against-non-consumption: substitutes and do-nothing are
  competitors, often the real ones.

## Good signals

- Three competitor classes are mapped explicitly: direct, substitute,
  do-nothing.
- The do-nothing alternative is named with the cost users currently
  absorb to not solve the problem.
- Differentiation is a tradeoff (small + fast + opinionated vs large
  + broad + flexible) competitors can't or won't copy — not a
  feature delta.
- Moats are evaluated by source (brand, ecosystem, data, switching
  cost, network effects, regulatory, scale economics) and each has
  a rough strength rating.
- Pricing comparison includes value-anchor, not just dollar amount
  ("$50/user/mo unlocks $400/month of saved labor at 5 hr × $20/hr").
- Positioning competitors hold themselves to is named (per Dunford —
  what reference frame does each competitor make the buyer use).

## Common failures

- **Only mapping direct competitors.** The convo's named failure: the
  real competition is usually the spreadsheet / email / manual
  workaround. Mitigation: mandatory do-nothing column.
- **Feature-list differentiation.** "Better, faster, cheaper" without
  a structural tradeoff competitors can't make. Differentiation is a
  decision, not an adjective.
- **Moat by hope.** "Once we have users, we'll have data, which is a
  moat." Data moats need named cycles (data → product improvement →
  more users → more data) and a measurable feedback loop.
- **Positioning that describes the product, not the buyer's
  reference frame.** "We're the AI-powered X for Y" without naming
  what X gets compared against. The buyer's reference frame
  determines whether your message lands.
- **Competitor map without funding / runway / urgency.** A
  well-funded competitor with 18 months of runway is a different
  threat than a 4-person team with 6.
- **Status quo treated as 'no competition'.** No competition usually
  means no market.

## Heuristics

- **(scope, investigate)** *Three competitor classes mandatory.*
  Direct, substitute, do-nothing. The do-nothing column has the
  current cost the user absorbs to not buy.
- **(investigate)** *Differentiation as tradeoff.* Name what we are
  worse at than each competitor. If we can't, we don't have
  differentiation — we have wishful thinking.
- **(investigate, decide)** *Score moats by source and strength.*
  Brand, ecosystem, data, switching cost, network effects,
  regulatory, scale economics. Each L / M / H with the cycle that
  produces it.
- **(investigate)** *Name positioning per competitor.* What
  reference frame does each competitor make the buyer use? Your
  positioning fights *for that frame*, not against features.
- **(investigate)** *Funding / runway / urgency.* For each direct
  competitor: last round, employee count, public roadmap if any,
  what they're prioritizing.
- **(decide)** *Survive the do-nothing test.* If the customer can
  reasonably continue what they're doing now, our positioning must
  beat that — features alone won't.
- **(scope, decide)** *Sub-segment monopoly check.* Is there a
  segment small enough that we can dominate it in 12 months? If yes,
  beachhead there; if no, the market is rivalry-dominated and the
  bet changes shape.

## Quick diagnostic

| Question | If no | Action |
| --- | --- | --- |
| Are direct, substitute, and do-nothing competitors mapped? | Direct-only map | Add substitute + do-nothing columns. |
| Is differentiation a structural tradeoff, not a feature delta? | Feature differentiation | Name what we're worse at; if we can't, we don't have differentiation. |
| Are moats sourced and rated? | Vague moats | Score by source × strength; name the cycle. |
| Is each competitor's positioning (reference frame) named? | Product-described, not buyer-reference-framed | Re-do positioning analysis from buyer-perspective. |
| Is funding / runway / urgency captured for direct competitors? | Static map | Pull last-round, employee count, public roadmap. |
| Does positioning survive the do-nothing comparison? | "Better features" only | Re-position against the workaround, not the named competitor. |
