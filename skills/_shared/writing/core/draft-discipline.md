# Draft Discipline

The get-words-down method. Used by `draft`. Defeats the blank-page stall that
keeps PRDs, specs, and posts unwritten.

## Shrink the task

Reduce the work to the **next small unit** — one section, one paragraph, one
example — instead of "the whole document." A frame small enough to finish in
one sitting is one you'll actually start. Deliver the piece one manageable
unit at a time.

## Separate generating from editing

Produce a **deliberately rough first pass** without judging it. The first
draft exists to surface the thought, not to be good; it's raw material every
later draft is built from. Don't polish a sentence at the moment of writing
it — that's the fastest way to stall. Generate now, edit later (hand the
result to [`clarity-rubric.md`](./clarity-rubric.md) on the revise pass).

## Set a stop

Give the draft a scope and an endpoint so it doesn't sprawl. "Rough first
pass of the three problem statements" is a finishable assignment; "the doc"
is not.

## What this is not

This is about overcoming perfectionism and paralysis, not a license to ship
the rough pass. The discipline is *draft → then revise*. If the ask is to
polish or tighten existing text, that's the `revise`/`copyedit` work, not
this.
